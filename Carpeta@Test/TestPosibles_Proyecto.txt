@Tests Posibles

Nota: Para los  @Tests podemos utilizar la clase ConcesionarioExcepcion.java
que tenemos.

@Test para Vehículos.

@Test añadir vehículo() -> Método que nos dé el número de vehículos que hay
                            actualmente y que al añadirle uno aumenta la
                            cantidad de vehículos actuales.

@Test eliminar vehículo() -> Como añadir vehículo pero reduciendo la unidad.

@Test modificación vehículo() -> Que acepte la modificación siempre y cuando
                                se aporten los datos necesarios para ello.



@Test filtro() -> Que al filtrar con un determinado parámetro, salga lo esperado.


@Test de inicio sesión() -> Cuando introducimos los datos del usuario, estos
                            se comprueben en la BD y salga positivo en caso
                            de existir.


@Test de no_inicio sesión() -> Cuando introducimos los datos del usuario, estos
                            se comprueben en la BD y salga positivo en caso
                            de no existir.



@Test conexión_BD() -> Test para comprobar si hay una conexión estable
                        con la base de datos.
